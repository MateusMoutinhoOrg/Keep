# Contribution Rules

Rules to follow when contributing to this project. Every file must also be shaped by the specification that governs it — locate it in [Specs.md](/docs/Reference/Meta/Rules/Specs.md).

---

## Specification Compliance

Before creating or editing any file, read [Specs.md](/docs/Reference/Meta/Rules/Specs.md) and check whether the file matches an **Applies To** entry. If it does, follow the specification that entry points to.

---

## File Changes

Before creating, deleting, or renaming any file or directory, read [Structure.md](/docs/Reference/Meta/Rules/Structure.md) and check whether the change affects the project structure. If it does, update [Structure.md](/docs/Reference/Meta/Rules/Structure.md) in the same commit.

---

## Documentation Changes

When you create, delete, or rename a `.md` file, update the Doc Index of [README.md](/docs/Reference/README.md).

---

## Sample Changes

When you create, delete, or rename a sample (any file inside [examples/](../../examples)), update the Samples section of [README.md](/docs/Reference/README.md).
